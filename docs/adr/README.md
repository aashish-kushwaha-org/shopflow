# What an ADR (Architecture Decision Record) actually is:

A short doc capturing what you decided, why, and what alternatives you considered/rejected — written so a future teammate (or future-you) doesn't have to reverse-engineer your reasoning from the code alone. Minimal structure per decision:

## ADR Template

> **Decision:** Decision title
> **Context**: what problem were you solving?
>
> **Decision**: what did you choose?
>
> **Alternatives considered**: what else did you weigh, and why rejected?
>
> **Consequences**: what does this make easier/harder going forward?
